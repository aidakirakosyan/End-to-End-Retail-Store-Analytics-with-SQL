
📖 Overview

I created a retail store database and developed analytical queries to explore sales, customers, and store performance.
The project demonstrates schema design, normalization, and advanced SQL query logic. It currently focuses on the database structure and analytics.

🛠️ Skills Demonstrated

Database design & normalization: Structured tables for categories, products, customers, stores, purchases, and deliveries.
Constraints & integrity: Applied foreign keys, triggers, and indexes to enforce consistency.
Analytical SQL: Wrote queries using CTEs, Joins, aggregations, window functions, and segmentation logic.
Customer analytics: Implemented RFM‑style segmentation (e.g., VIP, inactive, new customers).

📊 Analytical Queries

The repository includes a library of queries such as:

Daily sales monitoring
New product tracking
Revenue per product and per store
Top‑selling products
Customer purchase counts and segmentation
Daily revenue trends
Average order value

Each query is fully commented to explain both the technical logic and the business use case.

🚀 Future Potential

Power BI integration: The database was designed to connect with BI tools for dashboards and visualization.
Synthetic data expansion: Sample data will be added to simulate realistic business scenarios.
Scalability: The schema supports extension into larger datasets and enterprise‑level analytics.
